发表于 2019-07-26 更新于 2019-08-05

OSI参考模型

OSI七层模型
应用层 能够产生网络流量的应用程序
表示层 加密、压缩
会话层 服务端和客户端建立的会话 netstat -n
传输层 可靠传输、不可靠传输、流量控制
网络层 选择最佳路径、IP地址编址
数据链路层 数据如何封装、添加物理层地址
物理层 物理标准（电压、接口标准）
—

TCP/IP

TCP/IP四层模型
5 应用层 传输数据单元PDU —应用层数据—
4 传输层 报文 H4（运输层首部）—应用层数据—
3 网络层 数据报（加IP地址->数据包） H3（网络层首部）-H4（运输层首部）—应用层数据—
2 数据链路层 数据帧（加MAC地址->数据帧） H2（链路层首部）-H3（网络层首部）-H4（运输层首部）—应用层数据— -T2（链路层尾部）
1 物理层 01101101101111000 011011011比特流01111000
—

开放系统信息交换的几个概念
实体（entity） 交换信息的硬件或软件进程
协议 (protocol) 控制两个对等实体通信的规则
服务 (service) 下层向上层提供服务，上层需要使用下层提供的服务来实现本层的功能
服务访问点 (SAP) 相邻两层实体间交换信息的地方
