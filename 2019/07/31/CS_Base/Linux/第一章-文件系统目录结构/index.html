#一、Linux目录结构

  1. 基本介绍
    • Linux的文件系统是采用级层式的树状目录结构，在此结构中的最上层是根目录“/”，然后在此目录下再创建其他的目录
      • /bin 存放着最经常使用的命令
      • /sbin 存放着管理员使用的系统管理程序
      • /home 存放普通用户的主目录，在Linux中每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的
      • /root 管理员用户的主目录
      • /lib 系统开机所需要的基本动态链接共享库，其作用类似于Windows里的DLL文件。几乎所有的应用程序都需要用到这些共享库
      • /lost+found 该目录一般为空，当系统非法关机后，这里就存放了一些文件
      • /etc 所有的系统管理所需要的配置文件和目录
      • /usr 用户的很多应用程序和文件都放在该目录下，类似于windows的program files目录
      • /boot 存放的是启动Linux时使用的一些核心文件，包括一些连接文件以及镜像文件
      • /proc 该目录是一个虚拟目录，它是系统内存的映射，刚问这个目录来获取系统信息
      • /srv service缩写，该目录存放一些服务启动之后所需要提取的数据
      • /sys 这是linux2.6很大的变化，该目录下安装了2.6内核中新出现的一个文件系统
      • /tmp 该目是用来存放一些临时文件
      • /dev 类似于windows的设备管理器，把所有的硬件用文件的形式存储
      • /media linux系统会自动识别一些设备，例如U盘、光驱等，当识别后linux会把识别的设备挂载到该目录下
      • /mnt 系统提供该目录是为了让用户临时挂载别的文件系统，我们可以将外部的存储挂载在/mnt/上
      • /opt 这是给主机额外安装软件所摆放的目录，如安装ORACLE数据库就可放到该目录下。默认为空。
      • /usr/local 这是另一个主机给额外安装软件所安装的目录，一般是通过编译源码的方式安装的程序
      • /var 这个目录中存放着不断扩充着的东西，习惯将经常被修改的目录存放在这个目录下，包括各种日志文件
      • /selinux（security-enhance） SELinux是一种安全子系统，它能控制程序只能访问特定文件
