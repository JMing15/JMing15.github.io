常用的网络命令

  1. Linux配置IP地址的方法
  • ifconfig：查看与配置网络命令状态
    例：ifconfig eth0 192.168.0.200 netmask 255.255.255.0 #临时配置eth0网卡的ip和子网掩码
  1. Linux网络配置文件
  • /etc/sysconfig/network-scripts/ifcfg-eth0：网卡配置文件（选项大写，小写不报错但是不生效，参数可以小写）
  • ifconfig or ip addres show：配置文件命令
  • /etc/sysconfig/network：主机名文件
  • /etc/susconfig/resolv.conf:DNS配置文件
  1. 常用的网络命令
  • ifconfig：查看与配置网络命令状态
  • hostname：主机名
  • ifdown/ifup：关闭与启动网卡
  • route -n：查看路由列表（可以查看网关）
  1. 网关和路由

  • 路由：

    不同网段数据转发
    路由选择

  • 网关：

    不同网段数据转发
    路由选择
    默认路由
    NAT转换

  • route -n：查看系统中的路由表信息

临时网关设置：
* 添加：route add default gw ip
* 删除：route del default gw ip
永久网关设置：
* /etc/sysconfig/network-scripts/ifcfg-eth0

  1. DNS配置命令

  • nslookup [主机名或IP]:域名解析测试命令
    配置文件：

    局部：/etc/sysconfig/network-scripts/ifcfg-eth0
DNS=ip
全局：/etc/resolv.conf
nameserver ip
相关配置文件：/etc/hosts

  • netstat：查看网络状态

    • -t：列出TCP协议端口
    • -u：列出UDP协议端口
    • -n：不是用域名与服务名，而使用IP地址和端
    • l：仅列出在监听状态网络服务
    • a：列出所有的网络连接
    • -antp
    • -tlun

  • ping：查看网络连接是否相同，底层是ICMP协议

    • -i：指定间隔时间
    • -c：指定ping的次数
    • -s：指定数据包的大小

  • traceroute：路由查看命令，测试从当前主机到目的主机之间经过的网络节点数，用于追踪数据包在网络上传输时的全部路径，它默认发送的数据包大小是40字节，默认使用ICMP协议

    • -p：使用UDP端口进行测试，默认端口为33434
    • -q 3：指定测试时发送的数据包个数（即测试数）
    • n：以IP的方式进行连接测试，避开DNS解析

  • arp：地址解析协议，将ip地址解析成MAC地址

    • -a：查看所有
    • -d：ip地址，删除某条ARP记录

  • nmap：网络探测扫描命令 #此命令默认未安装

    • -sP：探测某网段内有哪些主机是活的
    • -sT：探测某主机上开启了哪些TCP端口

网络服务

一、ssh安全远程管理

环境准备

1. 临时关闭防护功能
    - iptables -F   #清空防火墙规则
    - setenforce 0  #临时关闭SELinux
2. 永久关闭防护功能
    - chkconfig iptables off    #设置防火墙开机不自动启动
    - sed -i '7s/enforcing/disabled/' /etc/selinux/config   #永久关闭SELinux
#以上命令都需重启才能生效

用户密码验证

1. Linux主机之间的远程管理工具是ssh命令，可以直接使用ssh进行远程登录
2. 登录命令： ssh 用户名@IP地址 #ssh IP地址默认root登录
3. 离开命令:  exit

主机之间的秘钥对登录验证

1. 客户端生成秘钥对文件
    ssh-keygen -t rsa -b 2048
    -t 指定加密类型（rsa/dsa等）
    -b 指定秘钥对加密长度
    询问1：执行过程中询问保存位置，一般默认保存在当前用户的家目录下的.ssh/目录下
    询问2：具体对秘钥文件进行加密（若加密，则在调用秘钥文件时需要先登陆验证秘钥密码，登陆正确才能使用秘钥文件；若不加密，则秘钥文件可以直接被调用，整个登陆验证过程无需输入任何密码，即为免密登陆）
2. 将公钥文件上传至服务器端
ssh-copy-id 用户名@服务器IP地址
#该用户名和要用来登陆服务器的用户名一致

3. 客户端尝试登陆服务器
ssh 用户名@服务器IP地址
#秘钥对验证优先级大于账户密码验证

禁止使用密码登陆

1. 配置文件：/etc/ssh/sshd_config 将PasswordAuthentication的选项由yes改为no
2. service sshd restart

禁止使用root远程登陆

1. 配置文件：/etc/ssh/sshd_config 将PermitRootLogin的选项由yes改为no
2. service sshd restart

修改默认端口、限制ssh监听IP

1. 修改默认端口：ssh作为一个用来远程管理服务器的工具，需要特别的安全，默认情况下使用TCP的22端口，若不进行修改，很容易遭受攻击，因此一般需要修改端口，尽量修改一个高位端口（范围1~65535）
    配置文件：/etc/ssh/sshd_config ，将选项Port修改为端口59527（举例），然后service sshd restart
    ssh -p 端口 用户名@服务器IP

2. 限制ssh监听IP:有些服务器安全级别较高，不允许使用外网直接登录，只有通过局域网才能登录，可以设置一台能够被外网远程连接的机子，其他服务器通过该机进行远程连接。
    配置文件：/etc/ssh/sshd_config 将ListenAddress改为监听IP

ssh服务相关命令（scp和sftp）

1. scp：安全的远程文件复制命令
    scp是secure copy的简写，用于在Linux下进行远程拷贝文件的命令，类似的命令有cp，scp传输是加密的，所以传输速度会受到影响。

    scp 本地文件 用户名@服务器IP：目录
    例：scp /root/**.txt root@10.10.10.10:/temp（本地上传至服务器）
        scp root@10.10.10.10:/temp /root/**.txt（服务器下载至本地）
    -P 端口 #若端口不是默认22，需要使用此格式指定的端口

3. sftp：安全的文件传输协议
    sftp是secure FileTransferProtocol的简写。sftp与ftp有着几乎一样的语法和功能。由于这种传输方式使用了加密/解密结束，所与ftp相比更加安全，但是传输小弟比ftp低许多

    sftp 用户名@服务器IP

二、TCP Wrappers（简单防火墙）

TCP Wrappers简介

TCP Wrappers是一个工作在传输层的安全工具，对有状态连接（TCP）的特定服务进行安全监测并实现访问控制，界定方式是凡是调用libwrap.so库文件的程序就可以接受TCP Wrappers的安全控制。它的主要功能就是控制谁可以访问，常见的程序有rpcbind、vsftpd、sshd、telnet.

判断方式：
1. 查看对应服务命令所在位置 which sshd
2. 查看指定命令执行时是否调用libwrap.so文件 ldd /usr/sbin/sshd | grep libwrap.so

TCP Wrappers工作原理

以SSH为例，每当有ssh的连接请求时，先读取系统管理员所设置的访问控制文件，若符合要求，则会把该连接原封不动地转给ssh进程，由ssh完成后续工作；若这次连接发起的ip不符合访问控制文件中的设置，则会中断连接请求。

示意图：
            客户端请求
                ↓
           TCP Wrappers
                ↓
            ip是否被写入
          /etc/hosts.allow-----是-----> 允许
                |
                | 否
                ↓
            ip是否被写入
          /etc/hosts.deny-------是----> 拒绝
                |
                | 否
                ↓---------------------> 允许
1.优选查看hosts.allow，匹配即停止
2.允许个别，拒绝所有：hosts.allow文件添加允许的策略，hosts.deny文件添加all
3.拒绝个别，允许所有：hosts.allow文件为空，hostss.deny文件添加单个拒绝策略

TCP Wrapers的使用

TCP Wrappers的使用主要是依靠两个配置文件/etc/hosts.allow，/etc/hosts.deny，以实现访问控制，默认情况下，/etc/hosts.allow，/etc/hosts.deny都为空，即没有访问控制。

配置文件编写规则：
1. service_list@host:client_list
service_list：是程序（服务）的列表，可以是多个，使用逗号分隔
host设置允许或禁止他人从本机的哪个ip地址进入。若无，则代表本机的所有ip地址都不可进入
client_list：是访问者的地址，如果需要控制的用户较多，可以使用空格或，隔client_list格式举例：
    基于IP地址：192.168.88.1（具体IP） 基于网段192.168.88.（基于网段）
    基于主机名：www.google.com .google.com
    基于网络/掩码：192.168.0.0/255.255.255.0
    内置ACL。ALL（所有主机）、LOCAL（本地主机）

例：
1.拒绝单个IP使用ssh远程连接
配置文件:
hosts.allow：空
hosts.deny:sshd:192.168.88.20

2.拒绝某个网段使用ssh远程连接
配置文件:
hosts.allow：空
hosts.deny:sshd:192.168.88.

3.仅允许某个IP使用ssh远程连接
配置文件:
hosts.allow：sshd:192.168.88.20
hosts.deny:sshd:ALL
