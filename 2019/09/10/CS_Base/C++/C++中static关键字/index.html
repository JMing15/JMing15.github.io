C++中static关键字

发表于 2019-09-10

Static关键字在不同的使用情景下拥有不同的含义，我们可以在以下情景使用static关键字：

  1. static变量：函数中的变量、类中的变量
  2. static类成员：class对象、class函数

一、static变量

函数中的static变量

使用static声明的变量，其获得的空间将存在于整个程序的生存期。即使该函数被多次调用，该static变量只申请一次空间。并且当前函数static变量会在下一次调用该函数继续使用。

输出：

0 1 2 3 4

我们可以看到上面的程序中count变量使用static声明，count在首次调用函数demo时初始化并只初始化一次，再次调用demo时count可以继续使用。

Class中的static变量

static声明的变量只初始化一次，其分配的空间位于静态存储区，类对象共享同一个static变量。多个类对象不能拥有static变量的多个副本，因此static变量不能使用构造函数初始化。

输出：

错误 1 error LNK2001: 无法解析的外部符号 “public: static int demo::i” (?i@demo@@2HA)

上面的程序我们试图通过多个对象创建static成员变量i的多个副本，但最终创建失败。因此，类中的static成员变量应该通过类名::成员名显式初始化

1

二、Class中的静态成员

static类对象

与静态变量类似，静态对象声明后，其存在于整个程序的生命周期。首先以非静态对象为例。

输出：

Inside Constructor
Inside Destructor
End of main

上面的例子中，在if的作用域内声明非静态对象d。因此，当对象d被创建时构造函数被触发，当程序运行离开if的作用域，对象d被销毁并触发析构函数。
让我们尝试把d声明为静态对象。

Inside Constructor
End of main
Inside Destructor

我们可以发现程序输出的结果改变了，静态对象d的析构函数在main函数也就是程序运行结束被触发。产生这个结果的原因是静态对象的生命周期存在于整个程序运行期间。

static类成员函数

与static变量、static类成员变量类似，static类成员函数也不依赖于类对象。我们可以使用对象+”.”运算符调用静态成员函数，但依然建议使用类名+”::”作用域运算符调用静态成员函数。
注意：静态成员函数只允许访问静态成员变量或其他静态成员函数，不允许访问类的非静态成员函数及变量。
