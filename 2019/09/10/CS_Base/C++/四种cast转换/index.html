四种cast转换

发表于 2019-09-10 更新于 2020-07-08

static_cast:

以下是more effective c++内容：
static_cast基本拥有与C旧式转型相同的威力与意义，以及相同的限制。

dynamic_cast:

以下是more effective c++内容：
dynamic_cast用来执行继承体系中“安全的”向下转型或跨系转型的动作，也就是说可以利用dynamic_cast，将指向“base class objects的pointers或references”转型为“指向derived（或sibling base）class objects的pointers或references”，并得知是否转型成功，如果转型失败，会以一个null指针（当转型对象是指针）或一个exception（当转型对象是reference）表现出来。dynamic_cast只能用来协助你巡航与继承体系中，它无法应用在缺乏虚函数的类型上。

const_cast:

以下是more effective c++内容：
const_cast用来改变表达式中的常量性或易变性，其最常见的用途就是将某个对象的常量性去掉。

reinterpret_cast:

以下是more effective c++内容：
reinterpret_cast几乎总是与平台息息相关，所以reinterpret_cast不具移植性，reinterpret_cast最常用用途是转换“函数指针”。假设有一个数组，存储的是函数指针，有特定的类型：typedef void (*FuncPtr);
FuncPtr funcPtrArray[10];
假设由于某个原因，你希望将以下函数的一个指针放进funcPtrArray中：
int doSomething();

如果没有转型，不可能办到这一点，因为doSomething的类型与funcPtrArray所能接受的不同。funcPtrArray内各函数的返回值是void，但doSomething的返回值却是int：
funcPtrArray[0]=&doSomething; //error

使用reinterpret_cast，可以强迫编译器了解你的意图
funcPtrArray[0]=reinterpret_cast(&doSomething);
