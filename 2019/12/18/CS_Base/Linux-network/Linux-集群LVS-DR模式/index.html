ARP响应级别

arp-ingore

  • 0 只要本机配置有相应IP地址就响应
  • 1 仅在请求的目标地址配置在请求到达的网络接口上时，才给予响应

    ARP通告行为

    arp-announce
  • 0 将本机任何网络接口上的任何地址都向外通告
  • 1 尽可能避免向目标网络通告与其网络不匹配的地址信息表
  • 2 仅向目标网络通过与其网络相匹配的地址信息

实验步骤

  1. 实验环境：
    10.10.10.240：客户机
    10.10.10.11：负载路由器
    10.10.10.12：服务器1
    10.10.10.13：服务器2

  2. 具体步骤

  • 负载路由器及服务器关闭网卡守护进程：
    service NetworkManager stop #临时关闭网卡守护进程
    chkconfig NetworkManager off #永久关闭网卡守护进程

  • 负载路由器开辟网卡子接口
    cd /etc/sysconfig/network-scripts/
    cp -a ifcfg-eth0 ifcfg-eth0:0
    修改 ifcfg-eth0:0 IP为10.10.10.100
    service network restart
    修改 /etc/sysctl.conf: 增加如下

    net.ipv4.conf.all.send_redirects = 0

    net.ipv4.conf.default.send_redirects = 0
net.ipv4.conf.eth0.send_redirects = 0

    sysctl -p #刷新内核参数
    yum -y install ipvsadm

  • 服务器1、2
    service httpd start
    echo “this is server 1” >> /var/www/html/index.html #模拟网站
    curl localhost
