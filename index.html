前言

网络层关注的是如何将分组从源端沿着网络路径送达目的端

一、网络的两种服务 虚拟路和数据报服务

两种服务：网络层应该向运输层提供这样的服务？

  • 虚电路服务
    • 虚电路表示这只是一条逻辑上的连接，分组都沿着这条逻辑连接按照存储转发方式传送，而不是建立了一条物理连接
  • 数据报服务
    • 网络层向上提供简单灵活的、无连接的、尽最大努力交付的数据报服务

#一、vi和vim编辑器的使用

##普通模式、插入/编辑模式、命令行模式切换

  • vim xxx 进入普通模式
  • i或者a 普通模式进入编辑模式

  • ESC命令模式进入普通模式

##vi和vim快捷键

  • yy:拷贝当前行,Nyy:拷贝当前行下的N行
  • p:粘贴
  • dd:删除当前行,Ndd：删除当前行下的N行
  • 普通模式下[/目标文本]:在文件中查找某个单词
  • set nu/set nonu:设置/取消文件行号
  • G/gg：到达文件最末行/大道文件首行
  • u：撤销
  • 普通模式下[N+SHIFT+G]:光标移动到第N行

#二、开机、重启和用户登录注销

##开机、重启指令

  • shutdown -h now 立即关机
  • shutdown -h 1 1分钟后关机
  • shutdown -r now 立即重启
  • halt 关机，作用和上面一样
  • reboot 立即重启
  • sync 把内存的数据同步到磁盘

##用户登录注销指令

  • logout:注销
    • logout指令在图形运行级别无效，在运行级别3下有效

#三、用户管理

##添加用户

  • useradd [选项] 用户名:添加用户
  • useradd -d 指令目录 用户名：给新建的用户指定家目录
  • passwd 用户名:设置用户的密码
  • cd:切换目录（change directory）
  • mkdir 目录名:创建目录
  • userdel 用户名:删除用户
    • 删除用户但保留家目录:userdel 用户名
    • 删除用户及用户主目录:userdel -r 用户名

##查询用户信息指令

  • id 用户名：查询用户信息

##切换用户

  • su - 用户名：切换至目标用户名
    • 高级用户向低级用户切换不需要使用密码，反之则需要
    • exit：返回至原来的用户

##查看当前用户

  • whoami:查询当前用户

##用户组

  • 用户组：类似于角色，系统可以对有共性的多个用户进行统一的管理
  • groupadd 组名：新增组
    • useradd -g 用户组 用户名：添加用户同时添加组
  • groupdel 组名：删除组
  • usermod -g 用户组 用户名：修改用户的组

##用户和组的相关文件

  • /etc/passwd：用户（user）的配置文件，记录用户的各种信息
    • 每行的含义：用户名；口令；用户标识号；组标识号；注释性描述；主目录；登陆Shell
  • /etc/shadow：口令的配置文件
    • 每行的含义：登录名；加密口令；最后一次修改时间；最小时间间隔；最大时间间隔；警告时间；不活动时间；失效时间；标志
  • /etc/group：组（group）的配置文件，记录Linux包含的组的信息
    • 每行的含义：组名；口令；组标识号；组内用户列表

#一、Linux目录结构

  1. 基本介绍
    • Linux的文件系统是采用级层式的树状目录结构，在此结构中的最上层是根目录“/”，然后在此目录下再创建其他的目录
      • /bin 存放着最经常使用的命令
      • /sbin 存放着管理员使用的系统管理程序
      • /home 存放普通用户的主目录，在Linux中每个用户都有一个自己的目录，一般该目录名是以用户的账号命名的
      • /root 管理员用户的主目录
      • /lib 系统开机所需要的基本动态链接共享库，其作用类似于Windows里的DLL文件。几乎所有的应用程序都需要用到这些共享库
      • /lost+found 该目录一般为空，当系统非法关机后，这里就存放了一些文件
      • /etc 所有的系统管理所需要的配置文件和目录
      • /usr 用户的很多应用程序和文件都放在该目录下，类似于windows的program files目录
      • /boot 存放的是启动Linux时使用的一些核心文件，包括一些连接文件以及镜像文件
      • /proc 该目录是一个虚拟目录，它是系统内存的映射，刚问这个目录来获取系统信息
      • /srv service缩写，该目录存放一些服务启动之后所需要提取的数据
      • /sys 这是linux2.6很大的变化，该目录下安装了2.6内核中新出现的一个文件系统
      • /tmp 该目是用来存放一些临时文件
      • /dev 类似于windows的设备管理器，把所有的硬件用文件的形式存储
      • /media linux系统会自动识别一些设备，例如U盘、光驱等，当识别后linux会把识别的设备挂载到该目录下
      • /mnt 系统提供该目录是为了让用户临时挂载别的文件系统，我们可以将外部的存储挂载在/mnt/上
      • /opt 这是给主机额外安装软件所摆放的目录，如安装ORACLE数据库就可放到该目录下。默认为空。
      • /usr/local 这是另一个主机给额外安装软件所安装的目录，一般是通过编译源码的方式安装的程序
      • /var 这个目录中存放着不断扩充着的东西，习惯将经常被修改的目录存放在这个目录下，包括各种日志文件
      • /selinux（security-enhance） SELinux是一种安全子系统，它能控制程序只能访问特定文件

#一、数据链路层基本概念及基本问题

##基本概念

  1. 数据链路层的信道类型
    • 点对点信道：使用一对一的点对点通信方式。
    • 广播信道：使用一对多的广播通信方式。
  2. 链路和数据链路
    • 链路：一条点对点的物理线路段，中间没有任何其他的交换节点。
    • 数据链路： 除了物理线路外，还必须有通信协议来控制这些数据的传输。
  3. 数据链路层传送的是帧

##基本问题

  1. 封装成帧
    • 在一段数据的前后分别添加收不和尾部构成一个帧，确定帧的界限。
    • 首部和尾部的一个重要作用就是帧的定界

#三级模式、两级映射

  1. 概念

    • 三级模式：内模式（物理层）、概念模式（逻辑层）、外模式（视图）
    • 两级映射：外模式<->概念模式、概念模式<->内模式

  2. 三级模式、两级映射的优点

    • 逻辑数据独立性：应用程序不受DBMS数据逻辑结构变化的影响
    • 物理数据独立性：应用程序不受DBMS数据物理结构变化的影响

#汇编指令编程（基本指令）

##基本指令

  1. debug基本指令
    • R：查看、改变寄存器的内容(R 寄存器名称)
    • D：查看内存中的内容（D 段地址：偏移地址
      例: D 1000:0 查看起始地址为1000:0及后的128个内存单元的内容
      D 1000:9 查看起始地址为1000:9机后的128个内存单元的内容
      D 1000:0 9 查看起始地址为1000:0，结尾地址为1000:9的内存单元内容）
    • E: 改写内存单元的内容(E 内存起始地址 数据 数据 数据…)
    • U：将内存中的机器指令翻译成汇编指令
    • T：执行一条机器指令
    • A：以汇编指令的格式在内存中写入一条机器指令
